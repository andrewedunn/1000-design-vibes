What the heck is this?

The unnecessarily detailed explanation

How did I end up here?

I was working on a project and hated my design. I asked Claude to generate a few options. That worked well enough that I wondered: what if I described a design using a bunch of different dimensions (color theory, typography style, visual density, cultural influence, etc. ) and let Claude go nuts?

Turns out you can. And once you have a few options why not make a thousand?

Can I use these designs?

Sure. At the top of each design is supposed to be some instructions for AI Agents to interpret the design and apply the design to a project (sometimes the subagent misses and doesn't give instructions). If you happen to use one of these designs and it works pretty well, give me a shout at hey at dunn dot us. I don't claim to own any of thise.

The Pipeline

Here's how a design goes from random numbers to a real HTML file:

1

Generate a Manifest

Python picks random values for each of the 31 dimensions, creates a unique seed, and generates a creative name.

2

Spawn Subagents

Claude Code reads the manifest and launches parallel subagents. Each one takes a design spec and generates a complete HTML file.

3

Validate & Collect

Each design goes through validation before being moved to the final folder.

The Manifest

The manifest is a JSON file that describes what designs to generate. Each entry includes a unique seed, a generated name, and values for all 31 dimensions.

manifest.json (excerpt)
{
  "id": 42,
  "seed": "a7f3c2e91b4d",
  "name": "Phantom Glacier",
  "tagline": "Neo-brutalist design with Japanese minimalism",
  "dimensions": {
    "ui_paradigm": "brutalist",
    "design_era": "neo_brutalist",
    "density": "airy",
    "color_theory": "monochromatic",
    "color_mode": "dark_only",
    "functional_direction": "dashboard",
    ...
  }
}

The Subagents

Each design is generated by a Claude Code subagent running in parallel. The agent gets the design spec from the manifest plus a design guide with rules and requirements.

Running agents in parallel means a batch of 100 takes about the same time as 10 would sequentially. This is mostly so I don't have to wait around.

The math, if you're curious

35 dimensions ~250 possible values

With 35 dimensions and around 250 possible values, there are more combinations than I'll ever generate. 1000 is basically nothing.

What each design includes

Every generated HTML file is self-contained:

  • Complete CSS with custom properties for all design tokens
  • Full component library (buttons, cards, forms, navigation)
  • Typography specimens at all heading levels
  • Color palette showcase
  • Keyboard and swipe navigation to browse designs

Want to try this?

The code is on GitHub. I've only used this with a Claude Max subscription. One of these days I'll run with other models. 

# Generate a manifest with 20 designs
python design_vibes.py manifest --count 20 --name "my-batch"

"Generate designs for outputs/2026-01-07-my-batch"
