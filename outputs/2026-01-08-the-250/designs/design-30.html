Introduction

Stark Core is a flat design system that combines international sensibility with pop art energy. Built for nonprofit organizations and community-driven projects that need trustworthy, accessible documentation.

Why Stark Core?

This design system prioritizes clarity and accessibility over decoration. Every element serves a purpose, and every color choice reinforces trust.

Installation

Get started by installing Stark Core via your preferred package manager:

Bash 
npm install @starkcore/design-system
# or
yarn add @starkcore/design-system
# or
pnpm add @starkcore/design-system

Requirements

  • Node.js 16.x or higher
  • Modern browser with CSS Grid support
  • JavaScript enabled for interactive features

Quick Start

Import Stark Core into your project and start building. The design system includes all necessary styles and components:

JavaScript 
import { ThemeProvider, Button } from '@starkcore/react';

function App() {
  return (
    <ThemeProvider>
      <Button variant="primary">
        Get Started
      </Button>
    </ThemeProvider>
  );
}
JavaScript 
<script setup>
import { Button } from '@starkcore/vue';
</script>

<template>
  <Button variant="primary">
    Get Started
  </Button>
</template>
HTML 
<link rel="stylesheet" href="starkcore.css">

<button class="button button-primary">
  Get Started
</button>

Configuration

Customize Stark Core by modifying CSS custom properties. All design tokens are exposed as CSS variables:

CSS 
:root {
  --color-primary: #FF3366;
  --color-secondary: #FFD700;
  --color-tertiary: #00D4FF;
  --font-heading: 'Archivo Black', sans-serif;
  --font-body: 'Work Sans', sans-serif;
  --space-unit: 8px;
  --border-width: 3px;
}

Theme Modes

Stark Core supports both light and dark modes out of the box. Toggle between themes using the data-theme attribute:

JavaScript 
// Toggle theme
document.documentElement.setAttribute('data-theme', 'dark');

// Listen for system preference
const darkModeQuery = window.matchMedia('(prefers-color-scheme: dark)');
darkModeQuery.addEventListener('change', (e) => {
  const theme = e.matches ? 'dark' : 'light';
  document.documentElement.setAttribute('data-theme', theme);
});
Accessibility First

All color combinations meet WCAG AA standards for contrast. Dark mode automatically adjusts colors to maintain accessibility.

Styling

Stark Core follows the flat design paradigm with these principles:

  1. No shadows or gradients - Clarity through contrast and color
  2. Bold borders - 3px borders define all interactive elements
  3. High contrast - Always meet WCAG AA standards
  4. Pop art palette - Primary colors with strategic accents

Accessibility

Accessibility is fundamental to Stark Core's trustworthy design philosophy:

  • Keyboard navigation - All interactive elements are keyboard accessible
  • Focus indicators - Visible focus states on all controls
  • Semantic HTML - Proper use of headings, landmarks, and ARIA
  • Color contrast - Minimum 4.5:1 for body text, 3:1 for large text
  • Reduced motion - Respects prefers-reduced-motion
  • Touch targets - Minimum 44x44px for all interactive elements
Screen Reader Testing

Stark Core has been tested with NVDA, JAWS, and VoiceOver. All components include appropriate ARIA labels and roles.

Keyboard Shortcuts
